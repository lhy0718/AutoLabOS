# P4 External Pilot Roadmap

P4 converts the audit-first surface from seed demo readiness into a pilot-ready artifact intake and full-seed regression package.

## Priorities

1. Accept an explicit external artifact root without exposing local source paths in generated outputs.
2. Emit a normalized claim-evidence export next to the audit report.
3. Expand seed replay coverage from AGB-001/003/010 to AGB-001 through AGB-010.
4. Keep literature discovery audit findings separate from manuscript paper-readiness blockers.
5. Use pilot evidence, not generic interest, as the go/no-go signal.

## Non-Goals

- No broad autonomous scientist repositioning.
- No paper-ready-by-default claims.
- No weakening of claim ceilings, result-table discipline, figure audit, citation support, failed-run visibility, or review gates.
- No committed machine-local paths or private source names.

## Competitive Watch Triggers

Add implementation work only when a tracked competitor introduces one of these concrete signals:

- runnable domain skill libraries with execution routing
- benchmark-store or run-store onboarding that improves artifact submission
- citation hallucination checks with claim-level evidence
- method-evolution provenance graphs tied to artifacts
- explicit governance, HITL, reproducibility, or failed-run preservation contracts

Papers, README claims, or repository appearances alone are watch items, not implementation triggers.
